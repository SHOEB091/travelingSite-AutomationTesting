🔴 Root Problems in Your Automation
1. ❗ StaleElementReferenceException (Cabs – From field)

This happens because:

Goibibo re-renders DOM after click/typing
Your stored WebElement becomes invalid

👉 Example from your log:

stale element not found in the current frame
✅ Fix

Never reuse WebElement after page updates.

❌ Wrong:

WebElement from = driver.findElement(...);
from.clear();
from.sendKeys("Delhi");

✅ Correct:

By fromLocator = By.xpath("//input[contains(@placeholder,'From')]");

wait.until(ExpectedConditions.elementToBeClickable(fromLocator)).click();
WebElement from = wait.until(ExpectedConditions.visibilityOfElementLocated(fromLocator));
from.clear();
from.sendKeys("Delhi");

✅ Better Locators for Goibibo

🟢 HANDLE POPUPS (CRITICAL)

Goibibo ALWAYS shows overlays.

👉 If you don’t close them → NOTHING works.

✅ Add this utility
public void closePopupIfPresent() {
    try {
        WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(5));
        WebElement closeBtn = wait.until(ExpectedConditions.visibilityOfElementLocated(
            By.xpath("//span[@role='presentation'] | //span[contains(@class,'close')]")
        ));
        closeBtn.click();
        System.out.println("Popup closed");
    } catch (Exception e) {
        System.out.println("No popup");
    }
}

👉 Call this after opening site and after navigation.

🔵 FIX STALE ELEMENT PROPERLY

Use retry logic:

public void enterText(By locator, String value) {
    for (int i = 0; i < 3; i++) {
        try {
            WebElement element = wait.until(ExpectedConditions.elementToBeClickable(locator));
            element.clear();
            element.sendKeys(value);
            return;
        } catch (StaleElementReferenceException e) {
            System.out.println("Retrying due to stale element...");
        }
    }
}
